ISSUES FOUND

- **Issue 1: Missing evidence for one sub-check can overwrite proven failures with an `ADVISORY` score**
  - **Category:** Correctness
  - **Severity:** Major
  - **Failure scenario:** A typical routed review lacks authenticated GitHub PR data or production deployment records—the runner explicitly treats both as best-effort. If the repository simultaneously has a proven defect such as cross-module work without `touches`, ineffective CODEOWNERS coverage, lightweight tags, or invalid tag ancestry, the output instructions still mandate `ADVISORY`. Teams therefore receive an uncertainty score for an established workflow failure, materially weakening the reusable prompt’s primary purpose. This is probable because missing `gh` authentication and deployment evidence are normal local-review conditions.
  - **Details:** **Violation:** The task requires “per-principle scoring with cited evidence” for integration ownership and tag correctness, but the prompt says, for Principle 4, “if that data is unavailable, say so, **score ADVISORY**,” and for Principle 6, “with mechanics-only evidence … **score ADVISORY**.” These unconditional instructions conflict with the same principles’ definitions of demonstrable `FAIL` conditions. **Impact:** A known defect is mislabeled as merely unevidenced, potentially changing remediation priority and the team’s interpretation of workflow health. **Evidence:** Principle 4 independently defines cross-module changes without `touches` and deficient CODEOWNERS composition as “Bad”; Principle 6 independently defines lightweight tags, inconsistent schemes, and bad ancestry as “Bad.” None of those facts depends on PR-review or production-target evidence. **Location:** `docs/tutorials/module-team-hello-world-review-prompt.md`, Principles 4 and 6 and their output-format instructions. **Fix:** Say that missing PR/deployment evidence prevents a `PASS` but does not suppress a `FAIL` established by available evidence—for example, “score no better than ADVISORY unless another evidenced condition requires FAIL.”
