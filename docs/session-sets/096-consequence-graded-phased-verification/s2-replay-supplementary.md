ISSUES FOUND

- **Issue 1: The review cannot reliably audit completed session-set work after the tutorial deletes its branches**
  - **Category:** Completeness
  - **Severity:** Major
  - **Failure scenario:** A team follows the walkthrough’s normal cleanup process, which deletes every merged session branch, and later runs the companion review monthly or quarterly as instructed. If the repository uses squash/rebase merges, customized merge titles, or has pushed the relevant merge outside the 50-commit log window—all common configurations—the evidence no longer connects a merged PR’s changed files to its session-set name and `module:` stamp. A completed `greeter` set that improperly edited `services/clock/` can therefore escape Principle 3 and the cross-module portion of Principle 4, producing `ADVISORY` or favorable coaching instead of the required cited failure.
  - **Details:**
    - **Violation:** The prompt claims to review “the current state of this repository against the seven principles,” including: “For each session-set branch … analyze the changed files.” The tutorial simultaneously requires merged branches to be deleted and tells users to run this prompt afterward as a “graduation check.”
    - **Impact:** The reusable review loses the ability to detect one of its central target violations once teams practice the prescribed trunk hygiene. That materially undermines the deliverable’s objective of providing evidence-cited coaching and would change a reasonable merge decision.
    - **Evidence:** The runner constructs `branches` only from live `refs/heads` and `refs/remotes`, then gathers diffs only for those surviving refs. Its merged-PR query requests `number,title,author,files,reviews,reviewDecision` but omits `headRefName`, so PR files cannot reliably be attributed to `session-set/<slug>` and its module stamp after branch deletion. Meanwhile, Parts 8–9 explicitly delete each local branch, enable automatic remote deletion, and the final checklist requires no merged branches to remain.
    - **Fix:** Gather `headRefName`, `mergedAt`, and preferably `mergeCommit`/URL for merged PRs; map `session-set/<slug>` head refs to the corresponding specs; and evaluate each merged PR’s `files` under Principles 3 and 4. Use an explicit review period rather than an arbitrary 30-PR limit, and score historical discipline `ADVISORY` when authenticated PR metadata is unavailable.
