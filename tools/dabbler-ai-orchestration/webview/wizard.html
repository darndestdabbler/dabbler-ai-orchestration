Dabbler AI Orchestration

An AI-led workflow for delivering software through structured, reviewable sessions. This extension is your command center.

How it works

1
Create a project plan. Write or generate a Markdown plan describing your project's phases and features. Import it with Dabbler: Import Project Plan.
2
Decompose into session sets. Use Dabbler: Generate Session-Set Prompt to get a prompt that turns your plan into a sequence of focused session specs.
3
Run sessions. For each session set, copy the start trigger from the Session Sets view and paste it into your AI assistant. The AI runs, commits, and logs its work.
4
Review and repeat. The Session Sets view shows real-time state. Review changes, approve or iterate, and move to the next session.
5
Ship. When all session sets reach "Done," your feature is built, reviewed, and ready.

Choose adoption tier

The tier changes one thing only — whether the AI router makes metered API calls. Lightweight is router-off, not Python-off: both tiers use a .venv + dabbler-ai-router and share the same lifecycle. Full model: tier-model.md. You can switch a set later by flipping its tier: field.

Prerequisites

✓VS Code 1.85 or later (you're here, so ✓)
○Python ≥ 3.10 with dabbler-ai-router installed in a workspace .venv (both tiers)
○At least one API key exported: ANTHROPIC_API_KEY, OPENAI_API_KEY, or GEMINI_API_KEY (Full only — Lightweight makes no metered calls)
○For out-of-band verification: a path-aware review agent for copyable prompts (Claude Code, Codex, Cline, Cursor, or any AI assistant with file-reading tools)
○Git installed and on your PATH
Cost reality. This workflow is not free. Each session typically costs $0.01–$2.00 depending on model and effort, and verification farms work out to a second model on top of that. Larger projects spanning days or weeks can accumulate into the tens or low hundreds of dollars. The author finds the tradeoff worth it; you may not, especially if you're doing volunteer or open-source work without independent funding. Open Dabbler: Show Cost Dashboard early and often, set a personal budget, and stop if you're not comfortable with the rate of spend. Anthropic pricing →
Lightweight tier — no metered API spend. Lightweight projects make zero router API calls, so there's no per-call cost. You still install dabbler-ai-router and run the same start_session / close_session lifecycle as Full — the router is simply off. Verification happens per-set: paste a copyable review prompt into a second AI assistant (free if you already have a subscription elsewhere), run a dedicated different-engine verification session, or opt out.
